# 飞书 Todo 机器人 - AI 增强功能使用指南

## 🎯 新增功能概览

本次更新为飞书 Todo 机器人添加了三大 AI 增强功能：

1. **📸 图片识别** - 自动从截图中提取任务信息

## 📋 配置说明

### 1. 环境变量配置

在 `.env` 文件中添加以下配置：

```env
# AI 配置（OpenAI 兼容接口）
OPENAI_API_BASE_URL=https://nowcoding.ai/v1
OPENAI_API_KEY=sk-n2J4fnW0UJc0Rt98xIsYpanJubxi5xyIFvZEBAun3iZsgjZM
OPENAI_MODEL=gpt-5.4-mini

# 视觉模型（可选，留空则使用 OPENAI_MODEL）
OPENAI_VISION_MODEL=

# AI 功能开关
ENABLE_IMAGE_RECOGNITION=true      # 启用图片识别
ENABLE_TODO_OPTIMIZATION=true      # 启用智能优化
ENABLE_AI_PARSE=true               # 启用 AI 增强解析
```

### 2. 配置说明

- **OPENAI_API_BASE_URL**: API 地址
  - 默认：`https://api.openai.com/v1`
  - 自定义：`https://nowcoding.ai/v1` 或其他 OpenAI 兼容接口
  
- **OPENAI_API_KEY**: API 密钥
  - 从你的 AI 服务提供商获取

- **OPENAI_MODEL**: 使用的模型
  - 示例：`gpt-5.4-mini`, `gpt-4`, `claude-3-opus` 等

- **OPENAI_VISION_MODEL**: 视觉模型（可选）
  - 如果你的 API 支持不同的视觉模型，可以单独指定
  - 留空则使用 `OPENAI_MODEL`

- **功能开关**: 可以单独控制每个 AI 功能的启用状态

## 🚀 功能使用

### 1. 📸 图片识别

**使用方法：** 直接发送图片给机器人

**支持的图片类型：**
- 会议通知截图
- 聊天记录截图
- 文档/笔记截图
- 任务清单截图

**示例：**

```
用户：[发送会议通知截图]

机器人：
🔍 正在识别图片中的任务信息...

✅ 已从图片中识别并创建 2 个任务：

1. 📝 产品评审会
   ⏰ 2026-05-28 14:00
   💡 3楼会议室，参与人：张三、李四

2. 📝 准备评审材料
   ⏰ 2026-05-28 13:00
   💡 会议前完成
```

**工作原理：**
1. 机器人接收图片
2. 下载图片并转换为 base64
3. 调用 Vision API 识别内容
4. 提取任务信息（标题、时间、优先级、备注）
5. 自动创建到飞书任务清单
6. 回复确认信息

**注意事项：**
- 置信度低于 0.5 的任务会被跳过
- 如果图片中没有明确的任务信息，会提示"未识别到任务"

### 2. 🎯 智能优化

**使用方法：** 发送以下命令之一

```
优化
优化任务
优化待办
整理任务
整理待办
optimize
```

**功能：**
- ⚡ 优先级合理性分析
- ⏰ 时间冲突检测
- 🔨 任务分解建议
- 🔗 任务合并建议
- ⏱️ 完成时间预估
- 📊 工作负荷评估

**示例：**

```
用户：优化

机器人：
🤔 正在分析你的待办清单...

📊 任务分析完成

⚠️ 发现 3 个优化建议：

1. 🔴 时间冲突
   "客户会议"(14:00) 和 "团队周会"(14:30) 时间重叠
   💡 建议将团队周会改到 16:00

2. 🟡 优先级
   "提交周报"截止今天 18:00，建议改为高优先级
   💡 当前标记为普通，但截止时间临近

3. 🟢 任务分解
   "完成项目方案"任务较大，建议拆分
   💡 可拆分为：收集需求、撰写初稿、内部评审

📝 总体评估：今天有 8 个任务，预计需要 6.5 小时，工作量适中。
```

**工作原理：**
1. 获取所有未完成任务
2. 调用 AI 分析任务列表
3. 从多个维度评估（优先级、时间、分解、合并等）
4. 生成优化建议
5. 返回结构化的分析结果

### 3. 🧠 增强解析

**使用方法：** 正常发送任务描述（自动启用）

**改进点：**
- 更准确的时间理解
- 自动识别优先级关键词
- 提取备注信息
- 理解复杂的任务描述

**示例：**

```
用户：明天下午3点前完成项目方案初稿，记得附上数据分析，这个很紧急

机器人：
✅ 任务已创建

📝 完成项目方案初稿
⏰ 2026-05-28 15:00
🔥 优先级：高
💡 备注：附上数据分析

[完成] [延期一天]
```

**对比旧版：**
- 旧版：可能无法正确识别"明天下午3点前"
- 新版：准确解析为截止时间 15:00
- 旧版：可能忽略"很紧急"
- 新版：自动设置为高优先级
- 旧版：备注信息可能丢失
- 新版：完整提取"附上数据分析"

## 🔧 技术实现

### 架构设计

```
用户消息
    ↓
飞书服务器
    ↓
你的服务器 (接收 Webhook)
    ↓
判断消息类型
    ├─ 图片 → 下载图片 → Vision API → 创建任务
    ├─ "优化" → 获取任务列表 → AI 分析 → 返回建议
    └─ 文本 → AI 解析 → 创建任务
```

### 核心文件

- **src/ai.ts** - AI 客户端和提示词模板
- **src/app.ts** - 消息路由和处理逻辑
- **src/feishu.ts** - 飞书 API 调用和图片下载
- **src/config.ts** - 配置管理

### 提示词设计

所有 AI 提示词都在 `src/ai.ts` 中定义：

1. **TASK_PARSE_PROMPT** - 任务解析提示词
2. **IMAGE_ANALYSIS_PROMPT** - 图片识别提示词
3. **OPTIMIZE_PROMPT** - 智能优化提示词

可以根据需要自定义这些提示词以获得更好的效果。

## 🐛 故障排查

### 图片识别失败

**可能原因：**
1. API 不支持 Vision 功能
2. 图片格式不支持
3. 图片太大或太小
4. API 配额不足

**解决方法：**
- 检查 API 是否支持 `image_url` 类型
- 确认 `OPENAI_VISION_MODEL` 配置正确
- 查看服务器日志获取详细错误信息

### 智能优化无响应

**可能原因：**
1. 任务列表为空
2. API 超时
3. 功能未启用

**解决方法：**
- 确认 `ENABLE_TODO_OPTIMIZATION=true`
- 检查是否有未完成的任务
- 增加 API 超时时间

### AI 解析不准确

**可能原因：**
1. 提示词不适合你的使用场景
2. 模型能力不足
3. 输入格式特殊

**解决方法：**
- 修改 `src/ai.ts` 中的提示词
- 尝试更强大的模型（如 GPT-4）
- 提供更清晰的任务描述

## 💡 最佳实践

### 1. 图片识别

**推荐做法：**
- 截图清晰，文字可读
- 包含明确的时间信息
- 避免过于复杂的布局

**不推荐：**
- 模糊的照片
- 手写内容（识别率低）
- 纯图片无文字

### 2. 智能优化

**推荐频率：**
- 每天早上查看一次
- 任务较多时（>10个）使用
- 感觉任务混乱时使用

**配合使用：**
- 先优化，再开始工作
- 根据建议调整任务
- 定期回顾优化效果

### 3. 任务描述

**推荐格式：**
```
明天 3点 提交周报 p1 备注:要附上数据
下周一前 完成方案 紧急
```

**关键词：**
- 时间：明天、下周、3点、15:00
- 优先级：p1/p2/p3、紧急、重要、普通
- 备注：备注:、注意:、说明:

## 🔐 安全建议

1. **API 密钥保护**
   - 不要将 `.env` 文件提交到 Git
   - 使用环境变量管理敏感信息
   - 定期轮换 API 密钥

2. **图片隐私**
   - 图片会发送到 AI API 进行分析
   - 避免上传包含敏感信息的截图
   - 了解你的 API 提供商的数据政策

3. **成本控制**
   - 监控 API 使用量
   - 设置使用限额
   - 图片识别成本较高，注意控制

## 📈 性能优化

### 响应时间

- 文本解析：~1-2秒
- 图片识别：~3-5秒
- 智能优化：~2-4秒

### 成本估算（以 OpenAI 为例）

- 文本解析：~$0.0001/次
- 图片识别：~$0.01/张
- 智能优化：~$0.001/次

### 优化建议

1. 使用更快的模型（如 gpt-3.5-turbo）
2. 减少不必要的 AI 调用
3. 缓存常见的解析结果
4. 使用本地模型（如 Ollama）降低成本
