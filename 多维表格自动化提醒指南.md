# 飞书多维表格自动化 · 到期提醒配置指南

不写代码、零运维、不消耗 AI token 的方案。本指南教你用「多维表格自动化」配置三种提醒。

---

## 🎯 你将获得的能力

| 提醒类型 | 触发时机 | 收件人 |
|---------|---------|-------|
| 📅 **每日汇总** | 每天 09:00（节假日跳过） | 每位执行人 |
| 🔔 **到期提醒** | 截止当天 09:00 / 14:00 / 18:00 | 执行人 |
| ⏰ **逾期提醒** | 逾期当天 1 次（不重复打扰） | 执行人 |

---

## 第一步 · 准备表格字段（重要）

打开你的多维表格 https://feishu.cn/base/HEAHbiuUFaQhE7siqzic9vjvnpf

确认/添加以下字段：

| 字段名 | 类型 | 必填 | 说明 |
|--------|-----|-----|-----|
| 待办事项 | 文本 | ✅ | 任务标题 |
| 截止日期 | 日期 | ✅ | 用作触发条件 |
| 优先级 | 单选 | ✅ | 🔴P0-高优 / 🟡P1-一般 / 🟢P2-低优 |
| 是否已完成 | 复选框 | ✅ | 关键过滤字段 |
| 执行人 | 人员 | ✅ | 收件人，**必须填**才能收到提醒 |
| **已提醒** | 复选框 | ⭐ 新增 | 防止逾期重复提醒 |

> ⚠️ "已提醒" 是新字段，请手动添加。如果不加，逾期提醒会每次都发。

---

## 第二步 · 进入自动化中心

1. 打开多维表格
2. 右上角点击 **「⚡ 自动化」** 图标
3. 点击 **「+ 新建自动化」**

> 不同版本飞书 UI 略有差异，找不到就在右上角"…"菜单里找"自动化"。

---

## 第三步 · 自动化 1️⃣ ：每日 09:00 汇总

### 配置参数

| 项 | 值 |
|---|---|
| 自动化名称 | `每日待办汇总` |
| 触发器 | **定时触发** |
| 触发时间 | 每天 `09:00` |

### 条件（多个 AND）

```
是否已完成 = 否
AND 截止日期 ≤ 今天
AND 执行人 不为空
```

### 操作

选 **「发送飞书消息」**：
- 接收人：**字段值 → 执行人**（这样会自动发给每位负责人）
- 消息类型：富文本/卡片
- 内容模板：

```
☀️ 早安，今日待办（含逾期）

📋 {{待办事项}}
🔴 优先级：{{优先级}}
🗓 截止：{{截止日期}}
{{#if 备注}}📝 备注：{{备注}}{{/if}}

👉 点击查看详情：{{记录链接}}
```

> 飞书自动化支持 `{{字段名}}` 模板变量。具体语法请参考飞书自动化的"插入字段"按钮。

### 节假日跳过（可选）

在条件里加一行：
```
WEEKDAY(TODAY()) IN [1,2,3,4,5]
```
（1=周一, 7=周日，留下 1-5 即工作日）

> 法定节假日（春节、国庆等）：建一张「节假日表」，自动化里加查询条件"今天不在节假日表"。详见文末"进阶"。

✅ 保存并启用。

---

## 第四步 · 自动化 2️⃣：到期当天提醒

> 与每日汇总不同：这条只发**今天到期**的，不含逾期，针对性更强。

### 配置参数

| 项 | 值 |
|---|---|
| 自动化名称 | `到期当天提醒` |
| 触发器 | **定时触发** |
| 触发时间 | 每天 `09:00`、`14:00`、`18:00`（点 + 号添加多个） |

### 条件

```
是否已完成 = 否
AND 截止日期 = 今天
AND 执行人 不为空
```

### 操作

发送飞书消息给执行人：
```
🔔 到期提醒

📋 {{待办事项}} 今天截止！
🔴 优先级：{{优先级}}
🗓 截止：{{截止日期}}

请抓紧处理 💪
```

✅ 保存并启用。

---

## 第五步 · 自动化 3️⃣：逾期提醒（每条只一次）

### 配置参数

| 项 | 值 |
|---|---|
| 自动化名称 | `逾期提醒` |
| 触发器 | **定时触发** |
| 触发时间 | 每 2 小时一次（或每天 10:00） |

### 条件（关键：用「已提醒」字段去重）

```
是否已完成 = 否
AND 截止日期 < 今天
AND 已提醒 = 否          ← 这是防重复的关键
AND 执行人 不为空
```

### 操作（两步）

**Step A：发送飞书消息**给执行人
```
⚠️ 任务已逾期

📋 {{待办事项}}
🗓 原本应在 {{截止日期}} 完成
🔴 优先级：{{优先级}}

需要延期或标记完成吗？
```

**Step B：更新当前记录**
```
已提醒 → 勾选（true）
```

> 这个 Step B 是核心：勾上"已提醒"后，下一轮扫描时条件 `已提醒 = 否` 就排除掉了，**不会再发第二次**。

### 重置机制（可选）

如果用户**修改了截止日期**（即用户给了新 deadline），希望"已提醒"自动恢复未勾选：

新建第 4 个自动化：
- 触发：**字段变更 - 截止日期**
- 操作：**更新记录 - 已提醒 = 否**

这样改了 deadline，提醒就重新激活。

✅ 保存并启用。

## 第六步 · 验证

1. 创建一条待办，截止日期填**昨天**，执行人选自己
2. 等到下一个整点（或临时把自动化触发时间改成"现在 +2 分钟"）
3. 飞书机器人应该会私信你提醒
4. 检查"已提醒"字段是否被自动勾上 ✅
5. 等下一轮扫描，确认**不会再次提醒** ✅

## 常见问题

**Q：自动化没触发？**
- 确认自动化已 **启用**（开关在卡片右上角）
- 触发时间是飞书服务器时间（中国区是北京时间）
- 检查条件是否过严，可以临时把条件去掉测试

**Q：消息没发到我？**
- "执行人"字段必须填，且填的是飞书联系人
- 机器人需要有"发送消息"权限
- 你和机器人是否同租户

**Q：节假日怎么自动跳过？**
- 简单方案：条件加 `WEEKDAY(TODAY()) IN [1,2,3,4,5]` 跳过周末
- 完整方案：建一张「节假日表」每年初维护一次，自动化里加 LOOKUP 校验
- 临时跳过：直接关闭自动化开关

**Q：想发到群里而不是私聊？**
- "发送飞书消息" 操作里把"接收人"改成"指定群组"
- 适合团队待办看板

**Q：提醒太多被打扰？**
- 调整 `优先级 = 🔴P0-高优` 才发提醒
- 或减少触发频率（比如每天只 09:00 一次）

## 进阶：法定节假日跳过

### 方法：建立节假日表

1. 多维表格里**新建一张表**，命名 `holidays`
2. 字段：`日期`（日期）、`名称`（文本）
3. 录入当年法定节假日（如 2026-01-01 元旦、2026-02-17 春节...）

### 在自动化里加查询

在 "每日汇总" 的条件里追加：
```
NOT EXISTS(holidays WHERE 日期 = TODAY())
```

> 飞书自动化的 LOOKUP/EXISTS 语法可能需要查阅文档，最稳妥是先把"日期=今天"做成自动化里的"条件分支"。

### 替代方案：用本地服务

如果觉得维护节假日表麻烦，可以**保留我们写好的本地服务**做这件事：
- 本地服务订阅"今天是不是工作日"
- 工作日才发飞书 webhook 触发自动化

这就是混合方案，但要电脑常开。

## 和本地 Bot 的关系

这套自动化方案**完全独立运行**，和你已有的 AI Bot 互不冲突：

```
┌──────────────────────────────────┐
│  飞书多维表格                     │
│  ├─ 自动化提醒（本指南）          │ ← 不消耗 token
│  └─ 数据存储                      │
└────────────┬─────────────────────┘
             │
             ↓ 用户主动聊天时
┌──────────────────────────────────┐
│  本地 AI Bot                      │
│  └─ Agent + 工具调用              │ ← 此时才消耗 token
└──────────────────────────────────┘
```

提醒发出去后，用户回复"已完成"或者跟 Bot 聊"把 xxx 标记完成"，Bot 才会启动 AI 处理。
