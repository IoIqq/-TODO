# 🎯 飞书回调地址配置指南

## 📋 配置前检查清单

在配置飞书回调之前，请先确认：

✅ **本地配置完成**
- `.env` 文件已创建并填写所有必需配置
- 运行 `node check-config.js` 检查配置是否正确

✅ **服务正常运行**
- 本地服务已启动：`npm run dev`
- 健康检查通过：访问 `http://localhost:3000/health` 返回 "ok"

✅ **公网隧道已建立**
- Cloudflare Tunnel 或 ngrok 正在运行
- 获得了公网访问地址

---

## 🌐 获取公网地址

### 方案 A：Cloudflare Tunnel（推荐用于临时测试）

```bash
# 在 D:\cloud 目录或其他位置运行
cloudflared-windows-amd64.exe tunnel --url http://localhost:3000
```

启动后会显示类似：
```
https://louise-classical-yacht-forward.trycloudflare.com
```

⚠️ **注意**：这个地址在 cloudflared 重启后会变化，需要重新配置飞书回调。

### 方案 B：ngrok（推荐用于长期使用）

```bash
ngrok http 3000
```

免费版每次启动地址会变化，付费版可以获得固定域名。

### 方案 C：部署到云服务器（推荐用于生产环境）

参考 `DEPLOYMENT.md` 部署到阿里云 ECS 或其他云服务器，获得固定公网 IP。

## 🔧 飞书开放平台配置步骤

### 1. 访问飞书开放平台

打开：https://open.feishu.cn/app

找到你的应用（App ID: **cli_aa9ecd7a45f8dbed**）

---

### 2. 配置事件订阅 ⭐ 重要

进入 **事件订阅** 页面：

#### 2.1 填写请求地址

**请求地址 URL**：
```
https://你的公网地址/feishu/events
```

例如：
```
https://louise-classical-yacht-forward.trycloudflare.com/feishu/events
```

#### 2.2 验证 URL

点击"验证"按钮，系统会向你的服务器发送一个 challenge 请求。

**验证原理：**
- 飞书发送 POST 请求到你的 URL
- 请求体包含 `{ "challenge": "xxx", "token": "xxx", "type": "url_verification" }`
- 你的服务器需要返回 `{ "challenge": "xxx" }`
- 代码已实现（`src/app.ts` 中的 challenge 处理）

**验证成功标志：**
- ✅ 显示"验证成功"
- 如果失败，检查：
  - 本地服务是否运行（`npm run dev`）
  - 公网隧道是否正常
  - URL 路径是否正确（必须是 `/feishu/events`）
  - 查看本地终端日志

#### 2.3 订阅事件

确认已订阅以下事件：

- ✅ **im.message.receive_v1** - 接收消息
  - 用途：接收用户发送的消息，解析待办任务
  - 权限要求：`im:message`

- ✅ **card.action.trigger** - 卡片交互
  - 用途：处理卡片按钮点击（完成、延期）
  - 权限要求：`im:message`

点击"添加事件"可以添加更多事件。

#### 2.4 保存配置

点击页面底部的"保存"按钮。

---

### 3. 配置卡片回调

进入 **应用功能 → 机器人** 页面：

#### 3.1 找到消息卡片配置

在页面中找到"消息卡片请求网址"配置项。

#### 3.2 填写回调地址

**消息卡片请求网址**：
```
https://你的公网地址/feishu/events
```

⚠️ **注意**：卡片回调地址和事件订阅地址相同，都是 `/feishu/events`。

#### 3.3 保存配置

点击"保存"按钮。

---

### 4. 确认权限已开通 ⭐ 重要

进入 **权限管理** 页面，确认以下权限已开通：

#### 4.1 消息相关权限

- ✅ **im:message** - 获取与发送单聊、群组消息
  - 用途：接收用户消息、发送回复
  - 必需

- ✅ **im:message:send_as_bot** - 以应用的身份发消息
  - 用途：机器人主动发送消息
  - 必需

#### 4.2 任务相关权限

- ✅ **task:task** - 任务管理
  - 用途：基础任务操作
  - 必需

- ✅ **task:task:write** - 创建、删除、更新任务
  - 用途：创建待办、标记完成、延期
  - 必需

- ✅ **task:tasklist:read** - 读取任务清单
  - 用途：查询今天/明天的待办
  - 必需

#### 4.3 申请权限

如果权限未开通：
1. 点击对应权限右侧的"申请权限"按钮
2. 填写申请理由（例如："用于个人待办管理"）
3. 提交申请
4. 等待审核通过（自建应用通常立即通过）

## 🔍 故障排查指南

### 问题 1：URL 验证失败

**症状：**
- 飞书后台点击"验证"按钮失败
- 提示"请求超时"或"连接失败"

**排查步骤：**

1. **检查本地服务**
   ```bash
   # 访问健康检查端点
   curl http://localhost:3000/health
   # 应该返回: ok
   ```
   如果失败，运行 `npm run dev` 启动服务

2. **检查公网隧道**
   ```bash
   # 访问公网地址的健康检查
   curl https://你的公网地址/health
   # 应该返回: ok
   ```
   如果失败，重启 cloudflared 或 ngrok

3. **检查 URL 路径**
   - 确保 URL 以 `/feishu/events` 结尾
   - 不要有多余的斜杠或参数

4. **查看终端日志**
   - 本地终端应该显示收到 POST 请求
   - 如果没有日志，说明请求没到达服务器

---

### 问题 2：机器人不响应消息

**症状：**
- 在飞书中发送消息，机器人没有任何回复
- 终端没有收到请求日志

**排查步骤：**

1. **检查事件订阅配置**
   - 进入飞书后台 → 事件订阅
   - 确认 URL 验证成功（显示绿色✅）
   - 确认已订阅 `im.message.receive_v1` 事件

2. **检查权限**
   - 进入飞书后台 → 权限管理
   - 确认 `im:message` 和 `im:message:send_as_bot` 已开通

3. **检查机器人是否已添加**
   - 在飞书中搜索机器人名称
   - 确认已添加到对话或群聊

4. **查看飞书事件日志**
   - 进入飞书后台 → 事件订阅 → 事件日志
   - 查看是否有事件推送记录
   - 查看推送是否成功

5. **检查 Verification Token**
   - 确认 `.env` 中的 `FEISHU_VERIFICATION_TOKEN` 与飞书后台一致
   - Token 不匹配会导致请求被拒绝

---

### 问题 3：创建任务失败

**症状：**
- 发送待办消息后，机器人回复错误
- 终端显示 "Create task failed" 错误

**排查步骤：**

1. **检查任务清单 GUID**
   ```bash
   # 查看 .env 配置
   cat .env | grep FEISHU_TASKLIST_GUID
   ```
   - 确认 GUID 格式正确（UUID 格式）
   - 确认任务清单在飞书中存在

2. **检查任务权限**
   - 进入飞书后台 → 权限管理
   - 确认以下权限已开通：
     - `task:task`
     - `task:task:write`
     - `task:tasklist:read`

3. **测试 API 连接**
   - 查看终端日志中的详细错误信息
   - 常见错误码：
     - `99991663`: 权限不足
     - `99991401`: Token 无效
     - `230001`: 任务清单不存在

4. **验证 App ID 和 Secret**
   ```bash
   node check-config.js
   ```
   确认所有配置项正确

---

### 问题 4：卡片按钮无响应

**症状：**
- 点击卡片上的"完成"或"延期"按钮没有反应
- 没有 Toast 提示

**排查步骤：**

1. **检查卡片回调配置**
   - 进入飞书后台 → 应用功能 → 机器人
   - 确认"消息卡片请求网址"已配置
   - 地址应该是：`https://你的公网地址/feishu/events`

2. **检查事件订阅**
   - 确认已订阅 `card.action.trigger` 事件

3. **查看终端日志**
   - 点击按钮时，终端应该显示收到 POST 请求
   - 事件类型应该是 `card.action.trigger`

4. **检查卡片数据**
   - 卡片中应该包含 `task_guid` 和 `due_timestamp`
   - 如果缺少这些数据，按钮无法正常工作

---

### 问题 5：加密推送解密失败

**症状：**
- 终端显示 "Received encrypted payload but FEISHU_ENCRYPT_KEY is not set"
- 或者显示解密错误

**排查步骤：**

1. **检查加密配置**
   - 进入飞书后台 → 事件订阅
   - 查看是否启用了"加密推送"

2. **配置加密密钥**
   - 如果启用了加密，必须在 `.env` 中配置 `FEISHU_ENCRYPT_KEY`
   - 密钥应该与飞书后台显示的一致

3. **关闭加密（可选）**
   - 如果不需要加密，可以在飞书后台关闭
   - 关闭后需要重新验证 URL

---

### 问题 6：服务频繁重启或崩溃

**症状：**
- `npm run dev` 运行一段时间后自动退出
- 终端显示错误堆栈

**排查步骤：**

1. **查看错误日志**
   - 记录完整的错误堆栈
   - 查找关键错误信息

2. **常见错误原因**
   - 网络请求超时：检查网络连接
   - API 限流：降低请求频率
   - 内存不足：检查系统资源

3. **使用 PM2 管理进程（推荐）**
   ```bash
   npm install -g pm2
   pm2 start npm --name "feishu-todo" -- run dev
   pm2 logs feishu-todo
   ```

---

### 问题 7：公网地址频繁变化

**症状：**
- Cloudflare Tunnel 重启后地址变化
- 需要频繁更新飞书回调配置

**解决方案：**

**方案 A：使用 ngrok 付费版**
- 注册 ngrok 账号：https://ngrok.com/
- 购买付费计划获得固定域名
- 配置固定域名后无需再修改

**方案 B：部署到云服务器**
- 参考 `DEPLOYMENT.md` 部署到阿里云 ECS
- 获得固定公网 IP
- 一次配置，长期使用

**方案 C：使用 frp 自建隧道**
- 需要有公网服务器
- 配置 frp 服务端和客户端
- 获得固定的隧道地址

```bash
  "type": "url_verification"
}
```

应该返回：
```json
{
```
